# Layout Labo

## 概要

Layout Labo は、QGIS レイアウトで扱う地図アイテム情報を抽出、適用、再構成するためのプラグインです。

主に次の情報を扱います。

- 地図アイテムの Extent（`xmin`, `ymin`, `xmax`, `ymax`）
- `dataDefinedMapLayers.expression`

UI は単一ダイアログ内の複数タブ構成で、次の機能を提供します。

- エクスポート
- インポート
- 再作成
- 地図コピー
- expression ビルダ
- ページ考慮 zvalue 再設定

## 機能説明

### 1. エクスポート（既存レイアウト → CSV）

既存レイアウトから地図アイテム情報を抽出し、1 つの CSV に書き出します。

- 複数レイアウトを同時選択可能
- 出力 CSV は BOM 付き UTF-8
- 文字列列は CSV 上で引用符付きで出力
- 数値は不要な丸めをせずに出力

最短手順:

1. エクスポートタブで対象レイアウトを選択
2. 出力先 CSV を指定
3. `エクスポート実行` を押下

### 2. インポート（CSV → 既存レイアウト）

CSV の内容を既存レイアウトへ適用します。新規レイアウトは作成しません。

- 適用対象は `layout_name` 完全一致のみ
- 地図アイテム対応は `itemById()` を使用
- 一部不一致があっても継続可能なものは処理を継続

最短手順:

1. インポートタブで入力 CSV を指定
2. プロジェクト内の適用先レイアウトを選択
3. `インポート実行` を押下

### 3. 再作成（CSV + QPTテンプレート → 新規レイアウト）

CSV の情報を使い、QPT テンプレートからレイアウトを再作成します。

- 同名レイアウトが存在する場合は削除後に再作成
- 地図アイテム不一致などの警告が出たレイアウトは末尾に `_要確認` を付与
- 対応できる地図アイテムは継続して反映

最短手順:

1. 再作成タブで入力 CSV を指定
2. テンプレート（`.qpt`）を指定
3. CSV 内の再作成対象レイアウトを選択
4. `再作成実行` を押下

### 4. 地図コピー（任意の地図アイテム間コピー）

任意の元地図アイテムから取得した Extent や expression を、任意の既存レイアウト内の複数地図アイテムへ適用します。

- CSV を使わずに直接コピー可能
- `EXTENT` のみ、`expression` のみ、両方の適用を選択可能
- 同一 `map_item_id` が複数ある場合は出現順を含めて個別指定
- コピー先地図アイテム一覧には表示モードとして `既定` / `式` / `固定` を表示

最短手順:

1. 元レイアウトと元地図アイテムを選択して `取得` を押下
2. コピー先レイアウトとコピー先地図アイテムを選択
3. コピー対象を選んで `コピー実施` を押下

### 5. expression ビルダ

プロジェクト内レイヤから expression を組み立て、任意の既存レイアウト内の複数地図アイテムへ適用します。

- レイヤパネル順のレイヤ一覧から選択
- 並び順に応じて `|` 区切りの expression を生成
- 適用先は 1 回の実行につき単一レイアウト
- 適用先地図アイテム一覧には表示モードとして `既定` / `式` / `固定` を表示し、`式` の場合は内容も確認可能

最短手順:

1. 利用するレイヤを expression 対象へ追加し順序を調整
2. `expression 生成` を押下
3. 適用先レイアウトと地図アイテムを選択して `expression 適用` を押下

### 6. ページ考慮 zvalue 再設定

レイアウト内の全レイアウトアイテムを対象に、ページ順を考慮して zvalue を再設定します。ページアイテム自体は対象外です。

- 1 ページ目を最前面として再配置
- ページ内の既存重なり順は維持
- Layout Designer が開いている場合は表示更新を試行

最短手順:

1. 対象レイアウトを選択
2. `zvalue 再設定実行` を押下

## CSV仕様

CSV は本プラグイン自身が出力し、本プラグイン自身が読み込む固定仕様です。

ヘッダーは次の固定列です。

1. `layout_name`
2. `map_item_id`
3. `xmin`
4. `ymin`
5. `xmax`
6. `ymax`
7. `expression`

補足:

- `layout_name` / `map_item_id` / Extent 列は必須
- `expression` は空文字を許容
- 入出力の文字コードは BOM 付き UTF-8
- 必須列不足、数値変換失敗、重複（同一 `layout_name` + `map_item_id`）は読み込み失敗
- 一般的な任意 CSV との広範囲互換は目的としていません

## UIとログ

- UI は単一ダイアログのタブ切り替え方式
- レイアウト一覧や関連状態は初回表示・タブ切替・再読込・各処理後に再取得
- ダイアログ下部の共通ログ表示欄で `INFO` / `WARNING` / `ERROR` を確認可能
- 警告がある処理でも、継続可能な範囲は処理を継続します
- 再作成で警告が発生したレイアウトには `_要確認` が付与されます

## 動作環境

- QGIS 3.4 以上
- Python（PyQGIS）
