# Layout Labo

## 概要

このプラグインは、QGISレイアウトの地図アイテム情報をCSVで移送するためのツールです。
次の情報を対象にします。

- 地図アイテムの Extent（`xmin`, `ymin`, `xmax`, `ymax`）
- `dataDefinedMapLayers.expression`

単一ダイアログ内の3タブ構成で、以下の機能を提供します。

- エクスポート
- インポート
- 再作成

## 機能説明

### 1. エクスポート（既存レイアウト → CSV）

既存レイアウトから地図アイテム情報を抽出し、1つのCSVに書き出します。

- 複数レイアウトを同時選択可能
- 出力CSVは BOM付きUTF-8
- 文字列列はCSV上で引用符付きで出力
- 数値は不要な丸めをせずに出力

操作手順:

1. エクスポートタブで対象レイアウトを選択
2. 出力先CSVを指定
3. `エクスポート実行` を押下

### 2. インポート（CSV → 既存レイアウト）

CSVの内容を、既存レイアウトへ適用します（新規レイアウトは作成しません）。

- 適用対象は `layout_name` 完全一致のみ
- 地図アイテム対応は `itemById()` を使用
- 一部不一致があっても継続可能なものは処理を継続

操作手順:

1. インポートタブで入力CSVを指定
2. プロジェクト内の適用先レイアウトを選択
3. `インポート実行` を押下

### 3. 再作成（CSV + QPTテンプレート → 新規レイアウト）

CSVの情報を使い、QPTテンプレートからレイアウトを再作成します。

- 同名レイアウトが存在する場合は削除後に再作成
- 地図アイテム不一致などの警告が出たレイアウトは末尾に `_要確認` を付与
- 対応できる地図アイテムは継続して反映

操作手順:

1. 再作成タブで入力CSVを指定
2. テンプレート（`.qpt`）を指定
3. CSV内の再作成対象レイアウトを選択
4. `再作成実行` を押下

## CSV仕様

ヘッダーは固定です。

1. `layout_name`
2. `map_item_id`
3. `xmin`
4. `ymin`
5. `xmax`
6. `ymax`
7. `expression`

補足:

- `layout_name` / `map_item_id` / Extent列は必須
- `expression` は空文字を許容
- 必須列不足、数値変換失敗、重複（同一 `layout_name` + `map_item_id`）は読み込み失敗

## UIとログ

- UIは単一ダイアログのタブ切り替え方式
- レイアウト一覧は初回表示・タブ切替・再読込・各処理後に再取得
- ログ表示欄で `INFO` / `WARNING` / `ERROR` を確認可能

## 動作環境

- QGIS 3.4以上
- Python（PyQGIS）
